# فلسفة التصميم - مشروع "سلامتك" (Salamatak)

## الرؤية الأساسية
تطبيق طبي ذكي يجمع بين الجمالية السيبرانية المستقبلية والدفء الإنساني، مما يحول التطبيقات الطبية الباردة إلى "تقنية ودودة" توفر الراحة والثقة للمريض.

## هندسة الروبوت التفاعلي (Cybernetic Assistant)

الروبوت ليس مجرد صورة، بل كائن برمجي حي يتفاعل مع المستخدم عبر حالات مختلفة:

### الحالات الأساسية

#### 1. **حالة السكون (Idle Mode)**
- عينا الروبوت تتحرك ببطء وتتبع حركة مؤشر الفأرة
- إيحاء بالانتباه والاهتمام بوجود المستخدم
- تأثير: `mouse-tracking` مع تأخير بسيط

#### 2. **حالة التحليل (Processing Mode)**
- ملامح الروبوت تتحول إلى حلقات رقمية دوارة
- تأثير `scanning` يشير إلى العمل الجاد والدقة
- الألوان تتحول إلى Neon Cyan

#### 3. **حالة التحدث (Speaking Mode)**
- موجات صوتية تظهر أسفل وجه الروبوت
- تتراقص متزامنة مع خروج صوت الذكاء الاصطناعي
- تأثير: `audio-visualizer` ديناميكي

#### 4. **حالة التنبيه (Alert Mode)**
- أطراف وجه الروبوت تومض باللون الأصفر أو الأحمر الهادئ
- عند رصد علامة حيوية مقلقة
- تأثير: `pulse` أو `glow` ناعم

## توزيع الواجهة (Bento Grid Layout)

### نظام Zero-Scrolling
عرض جميع المعلومات المهمة في شاشة واحدة دون الحاجة للتمرير:

```
┌─────────────────────────────────────────┐
│  الروبوت التفاعلي + شاشة المحادثة     │
│  (Hub AI - المركز الأكبر)              │
├──────────────────┬──────────────────────┤
│  القياسات الفورية│  سجل القياسات الأخير│
│  (Heart Rate)    │  (Recent Logs)       │
│  (Oxygen Level)  │  مع حذف/أرشفة سريعة│
├──────────────────┴──────────────────────┤
│  أزرار الإجراءات السريعة                │
│  (رفع ملف، بدء فحص، طلب طوارئ)        │
└─────────────────────────────────────────┘
```

## رحلة المستخدم السلسة (Frictionless Flow)

### قاعدة "الثلاث نقرات"

#### 1. **الفحص**
- المستخدم يضغط على زر "بدء الفحص"
- تظهر النتائج في ثوانٍ
- الروبوت يحللها تلقائياً

#### 2. **الاستشارة**
- المستخدم يرفع صورة التحليل (Drag & Drop)
- يقرأها نموذج Gemini 1.5 Flash
- الروبوت يشرحها بالصوت والصورة فوراً

#### 3. **الطوارئ**
- زر أحمر واضح في الزاوية دائماً
- عند الضغط: تظهر أهم أرقام طوارئ مع ملخص الحالة الصحية
- جاهزة للتقديم للمسعفين

## الحركة والتفاعل (Animation & Interaction)

### مبادئ الحركة
- **السرعة:** جميع الحركات تحت 300ms (معظمها 150-200ms)
- **الإيزينج:** استخدام `cubic-bezier(0.23, 1, 0.32, 1)` للخروج السريع
- **الأصول:** الحركات تبدأ من `scale(0.95)` مع `opacity: 0` (ليس من 0)
- **الاحترام:** احترام `prefers-reduced-motion` للمستخدمين الحساسين

### الحركات المحددة
- **دخول العناصر:** fade-in + slide-up (150ms)
- **تحويم الأزرار:** scale(1.05) + shadow increase (100ms)
- **نقر الأزرار:** scale(0.97) (160ms)
- **موجات الصوت:** تراقص متزامن مع الصوت (ديناميكي)

## قرارات التصميم

### الخلفيات
- الخلفية الرئيسية: Deep Space Blue مع تدرج ناعم نحو الأسود
- تأثيرات: شبكة رقمية ناعمة جداً في الخلفية (opacity: 5%)

### الحدود والفواصل
- حدود ناعمة: `border-radius: 12px` للبطاقات الرئيسية
- فواصل: خطوط رقيقة بـ Soft Gray مع opacity 30%

### الظلال
- ظلال ناعمة: `box-shadow: 0 4px 12px rgba(0, 255, 47, 0.1)` (Neon Cyan glow)
- ظلال عميقة: للعناصر المرفوعة والبطاقات
